우리동네스마트건강지도 서비스 요약

프로젝트 기반 발표 자료 (요청 순서 1~7 반영)

1. 서비스 명

우리동네스마트건강지도

지역 건강위험 정보와 증상 기반 병원 추천을 한 번에 제공하는 생활 밀착형 헬스케어 지도 서비스

2. 타겟(페르소나)

페르소나 A: 30대 직장인(성인)

  • 감기·몸살 등 증상 발생 시 빠른 병원 탐색 필요
  • 대기시간이 긴 병원은 피하고 싶음
  • 출퇴근 동선 내 가까운 병원 선호

페르소나 B: 40대 보호자(소아 동반)

  • 아이 증상에 맞는 진료과를 빠르게 파악하고 싶음
  • 야간/주말 혼잡도가 낮은 병원 탐색 필요

페르소나 C: 60대 이상 사용자(고령)

  • 복잡한 탐색 없이 간단 입력으로 추천받고 싶음
  • 이동 가능 거리 내 병원 우선 확인 필요

3. 특징

  • 지역 단위 건강위험수치 제공(시/군/구, 읍/면/동)
  • 증상 입력 시 추천 진료과 자동 반영
  • 병원 추천 버튼 한 번으로 주변 병원 후보 조회
  • 거리·예상 대기시간·진료과를 함께 고려한 추천
  • 실시간 조회 실패 시 지역 실병원 데이터 폴백

4. 핵심기능

  • 지역 건강위험수치 및 위험등급 표시
  • 건강예방 조언 자동 제공
  • 병원 대기시간 확인(추정/수집 정보 기반)
  • 증상/상황 기반 병원 추천(최대 5곳)
  • 감염 위험 알림 토글
  • 원격 대기 등록(데모 흐름)

5. 편리한 점

  • 진료과를 직접 고르지 않아도 증상만 입력하면 자동 추천
  • 이동 가능 거리 설정으로 현실적인 병원 후보만 제시
  • 한 화면에서 정보 조회와 추천 비교 가능
  • 거리/대기시간/진료과를 빠르게 비교 가능

6. 장점

  • 사용자의 병원 선택 의사결정 시간 단축
  • 의료 접근성 향상(가까운 병원 우선 탐색)
  • 외부 API 연동 + 폴백 구조로 서비스 안정성 확보
  • MVP 단계에서도 실제 배포 및 즉시 사용 가능

7. 단점

  • 일부 병원 데이터/대기정보는 추정치 기반
  • 지역별 실병원 데이터 커버리지가 불균등
  • 외부 API 응답 및 쿼터 상태에 따라 정확도 변동 가능
  • 개인 건강기록 연동/예약/결제 기능은 미포함
